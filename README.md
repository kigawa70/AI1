# 📰 Gemini API 動的RSSニュースリーダー

本プロジェクトは、Gemini APIを使用してRSSフィードからリアルタイムのニュースを取得し、
ニュースの内容に基づいてAIがタグを生成して表示するWebアプリケーションです。

## 🤖 ロジックの説明 (カテゴリ変更から画面表示まで)

カテゴリ変更から画面表示に至るまでのロジックは、「クライアントサイド (JavaScript)」と「サーバーサイド (Node.js/Express)」の連携によって実現されます。

### クライアントサイドの動作

1.  **カテゴリ選択**: ユーザーが画面上のカテゴリボタン（例: テクノロジー）をクリックすると、対応するカテゴリキー (`tech` など) を引数に `fetchNews(categoryKey)` 関数が実行されます。
2.  **ローディング表示**: ニュース一覧エリアに「読み込み中...」のメッセージを表示し、ユーザーに処理中であることを伝えます。
3.  **APIリクエスト**: サーバーサイドの `/api/news/:categoryKey` エンドポイントに対し、選択したカテゴリキーを含むHTTP GETリクエストを非同期で送信します。

### サーバーサイドの動作 (Node.js/Express)

1.  **リクエスト受付**: サーバーはクライアントからの `/api/news/:categoryKey` リクエストを受け取ります。
2.  **RSS取得**: `rss-parser` ライブラリを使用し、カテゴリに対応するRSS URLから最新のフィードを取得・パースします。
3.  **タグ生成 (Gemini APIの利用)**:
    * 取得したニュースアイテム（上位10件）をループ処理します。
    * 各ニュースのタイトルと概要を組み合わせたプロンプトを **Gemini API (`gemini-2.5-flash`)** に送信します。
    * Gemini APIは、プロンプトに基づき、ニュース内容を要約・分類する5つのキーワード（AIタグ）を生成します。
    * **※このGemini APIとの通信に時間がかかるため、課題の仕様として許容されています。**
4.  **レスポンス送信**: AIタグが付与されたニュースオブジェクトの配列をJSON形式でクライアントに返します。

### 再びクライアントサイド

1.  **データ取得**: サーバーからのJSONレスポンスを受け取ります。
2.  **画面更新**: 取得したニュースデータ（タイトル、リンク、**AIタグ**など）を基にHTML要素を動的に生成し、ニュース一覧エリアを更新します。

## 🚀 起動手順とインストール作業

### 必要な環境

* Node.js (バージョン 18以上を推奨)
* GitHubアカウント
* Gemini API Key

### インストールと起動

1.  **リポジトリをクローン**:
    ```bash
    git clone [あなたのGitHub URL]
    cd [プロジェクトディレクトリ名]
    ```

2.  **依存関係のインストール**:
    ```bash
    npm install
    ```

3.  **APIキーの設定**:
    プロジェクトのルートディレクトリに `.env` ファイルを作成し、あなたのGemini APIキーを設定します。
    ```
    # .env
    GEMINI_API_KEY="YOUR_API_KEY_HERE"
    ```

4.  **サーバーの起動**:
    ```bash
    node server.js
    ```
    (サーバーが `http://localhost:3000` で起動します)

5.  **Webブラウザでアクセス**:
    ブラウザで `http://localhost:3000` にアクセスしてください。
